## Rotas e conteúdo estático no Next.js 16 (App Router)

> **Versão de referência: `next@16.2.4`.**

O App Router usa **roteamento baseado em arquivos** dentro da pasta `app/`. Cada **pasta** dentro de `app/` define um **segmento de URL**. Para uma rota se tornar pública, é preciso adicionar um `page.js` (ou `route.js` para APIs).

### Arquivos especiais do App Router

| Arquivo                      | Função                                                               |
| ---------------------------- | -------------------------------------------------------------------- |
| `layout.js`                  | UI compartilhada pelos segmentos filhos (preserva estado ao navegar). |
| `page.js`                    | Torna a rota pública; renderiza a UI daquele segmento.               |
| `loading.js`                 | UI de carregamento (React `Suspense` automático).                    |
| `error.js`                   | Error boundary do segmento.                                          |
| `not-found.js`               | UI de 404 do segmento.                                               |
| `global-error.js`            | Error boundary raiz.                                                 |
| `route.js`                   | Route Handler (API) — substitui `pages/api`.                         |
| `template.js`                | Como `layout.js`, mas **remonta** a cada navegação.                  |
| `default.js`                 | Fallback de rotas paralelas.                                         |

Também existem convenções de pastas:

- `[slug]` — segmento dinâmico.
- `[...slug]` — catch-all.
- `[[...slug]]` — catch-all opcional.
- `(grupo)` — route group (não aparece na URL).
- `_privada` — pasta privada (não vira rota).
- `@slot` — rota paralela nomeada.

```mermaid

    A --> P0[page.js \"/\"]:::file

    A --> Api[api/]:::f
```

### 1. Rota padrão

Cada `page.js` vira uma rota. Para criar `/about`:

```bash
mkdir -p app/about
```

```jsx
// app/about/page.js
export default function AboutPage() {
  return (
    <main style={{ padding: 24 }}>
      <h1>Sobre nós</h1>
    </main>
  );
}
```

Acesse `http://localhost:3000/about`.

### 2. Rotas aninhadas e `layout.js`

Crie uma área `/blog` com layout próprio e uma home de blog.

```bash
mkdir -p app/blog
```

```jsx
// app/blog/layout.js
export default function BlogLayout({ children }) {
  return (
    <section style={{ maxWidth: 720, margin: '0 auto', padding: 16 }}>
      <nav style={{ marginBottom: 16 }}>
        <strong>Blog</strong>
      </nav>
      {children}
    </section>
  );
}
```

```jsx
// app/blog/page.js
export default function BlogHome() {
  return <h1>Blog — Página Inicial</h1>;
}
```

Adicione uma sub-rota `/blog/post`:

```bash
mkdir -p app/blog/post
```

```jsx
// app/blog/post/page.js
export default function Post() {
  return <h1>Post do Blog (rota /blog/post)</h1>;
}
```

> Ao navegar entre `/blog` e `/blog/post`, o `BlogLayout` **não remonta**: seu estado e UI são preservados.

### 3. Rotas dinâmicas com `[id]` e `params` assíncrono

> **Mudança importante (Next 15/16)**: `params` é uma **Promise** e precisa de `await`.

```bash
mkdir -p app/blog/\[slug\]
```

```jsx
// app/blog/[slug]/page.js
export default async function BlogPostPage({ params }) {
  const { slug } = await params;

  return (
    <article>
      <h1>Exibindo post: {slug}</h1>
    </article>
  );
}
```

Acessar `http://localhost:3000/blog/meu-primeiro-post` mostra o slug na página.

Para **pré-gerar** rotas estáticas (SSG), exporte `generateStaticParams`:

```jsx
// app/blog/[slug]/page.js
export async function generateStaticParams() {
  return [{ slug: 'primeiro' }, { slug: 'segundo' }];
}

export default async function BlogPostPage({ params }) {
  const { slug } = await params;
  return <h1>Post: {slug}</h1>;
}
```

### 4. Passagem de valores entre rotas: `<Link>` e `searchParams`

O componente `<Link>` vem de `next/link` e funciona no App Router com as mesmas propriedades de sempre, além de prefetch automático.

Página que envia um parâmetro via query string:

```jsx
// app/page.js
import Link from 'next/link';

export default function Home() {
  return (
    <main style={{ padding: 24 }}>
      <h1>Home</h1>
      <Link href="/about?name=João">Ir para Sobre com o nome "João"</Link>
    </main>
  );
}
```

Leia `searchParams` na página `/about` — também é uma **Promise**:

```jsx
// app/about/page.js
export default async function AboutPage({ searchParams }) {
  const { name } = await searchParams;
  return <h1>Sobre {name ?? 'desconhecido'}</h1>;
}
```

> **Importante**: usar `searchParams` faz o Next.js renderizar a página **dinamicamente** em cada requisição (pois depende da URL). Se quiser manter a rota pré-renderizada e ler query strings só no cliente, use o hook `useSearchParams` em um Client Component.

### 5. Navegação programática: `useRouter` de `next/navigation`

> **Importante**: no App Router, `useRouter` vem de `next/navigation` — **não** de `next/router` (este é exclusivo do Pages Router legado).

```jsx
// app/_components/LoginButton.js
'use client';

import { useRouter } from 'next/navigation';

export default function LoginButton() {
  const router = useRouter();
  return (
    <button onClick={() => router.push('/about?name=Maria')}>
      Entrar como Maria
    </button>
  );
}
```

Outros métodos úteis do `router`:

- `router.replace(url)` — navega sem adicionar ao histórico.
- `router.back()` / `router.forward()` — navegação de histórico.
- `router.refresh()` — re-busca os Server Components da rota atual.
- `router.prefetch(url)` — prefetch manual.

Para ler segments e pathname em Client Components:

- `usePathname()` — caminho atual.
- `useSearchParams()` — query string (síncrono no cliente).
- `useParams()` — parâmetros da rota dinâmica.

### 6. Listagem com detalhes (padrão lista → página dinâmica)

Estrutura:

```
app/
└── posts/
    ├── page.js        → "/posts"
    └── [id]/
        └── page.js    → "/posts/:id"
```

```jsx
// app/posts/page.js
import Link from 'next/link';

const posts = [
  { id: 1, title: 'Post 1' },
  { id: 2, title: 'Post 2' },
  { id: 3, title: 'Post 3' },
];

export default function PostsPage() {
  return (
    <main style={{ padding: 24 }}>
      <h1>Lista de Posts</h1>
      <ul>
        {posts.map((post) => (
          <li key={post.id}>
            <Link href={`/posts/${post.id}`}>{post.title}</Link>
          </li>
        ))}
      </ul>
    </main>
  );
}
```

Na página de detalhes, os dados normalmente viriam de uma API. Para simplicidade:

```jsx
// app/posts/[id]/page.js
import { notFound } from 'next/navigation';

const posts = [
  { id: '1', title: 'Post 1', content: 'Conteúdo do Post 1' },
  { id: '2', title: 'Post 2', content: 'Conteúdo do Post 2' },
  { id: '3', title: 'Post 3', content: 'Conteúdo do Post 3' },
];

export default async function PostDetails({ params }) {
  const { id } = await params;
  const post = posts.find((p) => p.id === id);

  if (!post) notFound();

  return (
    <article style={{ padding: 24 }}>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}
```

Adicione um `not-found.js` para customizar a página 404:

```jsx
// app/posts/[id]/not-found.js
export default function NotFound() {
  return (
    <main style={{ padding: 24 }}>
      <h2>Post não encontrado</h2>
    </main>
  );
}
```

### 7. Loading e Error UI automáticos

Suporte automático a estados de carregamento e erro por rota:

```jsx
// app/posts/loading.js
export default function Loading() {
  return <p>Carregando posts…</p>;
}
```

```jsx
// app/posts/error.js
'use client';

export default function Error({ error, reset }) {
  return (
    <div>
      <h2>Algo deu errado!</h2>
      <p>{error.message}</p>
      <button onClick={() => reset()}>Tentar novamente</button>
    </div>
  );
}
```

O `error.js` precisa ser Client Component (`'use client'`), pois usa event handler.

### 8. Route Handlers — APIs em `route.js`

No App Router, APIs são definidas com `route.js` dentro da pasta correspondente à URL. Exemplo de `GET /api/hello`:

```bash
mkdir -p app/api/hello
```

```js
// app/api/hello/route.js
export async function GET(request) {
  return Response.json({ message: 'Olá do Next.js 16!' });
}
```

Testando:

```bash
curl http://localhost:3000/api/hello
# {"message":"Olá do Next.js 16!"}
```

Suporta todos os métodos HTTP (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`). Usa `Request`/`Response` do padrão Web.

Exemplo com path params (`/api/posts/:id`):

```bash
mkdir -p app/api/posts/\[id\]
```

```js
// app/api/posts/[id]/route.js
export async function GET(request, { params }) {
  const { id } = await params;
  return Response.json({ id, title: `Post ${id}` });
}
```

> No Next 16, `params` continua sendo **Promise** em Route Handlers também.

### 9. Conteúdo estático na pasta `public/`

Qualquer arquivo dentro de `public/` é servido na raiz (`/`). Por exemplo:

```
public/
└── images/
    └── logo.png
```

Referencie com o caminho a partir de `/`:

```jsx
// app/page.js
import Image from 'next/image';

export default function Home() {
  return (
    <main style={{ padding: 24 }}>
      <h1>Bem-vindo</h1>
      <Image
        src="/images/logo.png"
        alt="Logo"
        width={200}
        height={60}
        priority
      />
    </main>
  );
}
```

Vantagens do `next/image`:

- Lazy loading por padrão (desative com `priority` para above-the-fold).
- Servir em formatos modernos (AVIF/WebP).
- `sizes` e `srcSet` automáticos.
- Exige `width`/`height` (ou `fill`) para evitar layout shift.

Para imagens **remotas**, configure o domínio em `next.config.mjs`:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactCompiler: true,
  images: {
    remotePatterns: [
      { protocol: 'https', hostname: 'images.unsplash.com' },
    ],
  },
};

export default nextConfig;
```

E use:

```jsx
<Image
  src="https://images.unsplash.com/photo-1503023345310-bd7c1de61c7d"
  alt="Foto remota"
  width={640}
  height={360}
/>
```

### 10. Fontes com `next/font`

O Next.js otimiza fontes em build (sem layout shift, sem request ao Google Fonts em runtime):

```jsx
// app/layout.js
import { Geist, Geist_Mono } from 'next/font/google';
import './globals.css';

const geist = Geist({ variable: '--font-geist', subsets: ['latin'] });
const geistMono = Geist_Mono({ variable: '--font-geist-mono', subsets: ['latin'] });

export default function RootLayout({ children }) {
  return (
    <html lang="pt-BR" className={`${geist.variable} ${geistMono.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

### 11. Proxy (antigo Middleware)

> **Next 16**: o arquivo `middleware.ts` foi renomeado para `proxy.ts` (a funcionalidade é a mesma).

Crie `proxy.js` na raiz (mesmo nível de `app/`):

```js
// proxy.js
import { NextResponse } from 'next/server';

export function proxy(request) {
  const isLogged = request.cookies.get('session')?.value;

  if (!isLogged && request.nextUrl.pathname.startsWith('/admin')) {
    return NextResponse.redirect(new URL('/login', request.url));
  }
  return NextResponse.next();
}

export const config = {
  matcher: ['/admin/:path*'],
};
```

Serve para reescritas, redirecionamentos, cabeçalhos e verificações rápidas antes da rota. **Não** use para lógica pesada ou fetch demorado.
