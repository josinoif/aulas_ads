## CRUD de produtos com Next.js 16 + crudcrud.com

> **Versões fixadas:** `next@16.2.4`, `react@19.2.4`, `axios@1.15.2`.

Neste tutorial vamos construir um CRUD completo de **produtos** com o App Router do Next.js 16. O backend é o [crudcrud.com](https://crudcrud.com/) — um serviço gratuito que dá uma URL temporária (expira em 24 h) com endpoints REST prontos.

A arquitetura terá:

- **Server Component** para listar produtos (buscando no servidor, zero JS no cliente para a lista).
- **Client Component** para o formulário e ações interativas.
- **Server Actions** para criar/editar/excluir e revalidar a listagem.

```mermaid
flowchart LR
    U[Usuário] -->|Navegar| SC[Server Component<br/>app/products/page.js]
    SC -->|fetch com cache tag| CC[crudcrud.com]
    U -->|Clicar \"Novo\"| FC[Client Component<br/>ProductForm]
    FC -->|chama Server Action| SA[Server Action<br/>createProduct / updateProduct / deleteProduct]
    SA -->|axios HTTP| CC
    SA -->|revalidateTag| SC
```

### Pré-requisitos

- Node.js ≥ 20.9
- npm ≥ 10
- Conta gratuita em [crudcrud.com](https://crudcrud.com/) (pega-se uma _unique key_ na home)

### Passo 1 — Criar o projeto

```bash
npx create-next-app@16.2.4 crud-products \
  --javascript \
  --eslint \
  --tailwind \
  --src-dir \
  --app \
  --turbopack \
  --import-alias "@/*" \
  --use-npm
```

Opções escolhidas:

- **`--src-dir`**: código dentro de `src/` (configuração solicitada originalmente).
- **`--app`**: App Router (padrão e recomendado).
- **`--turbopack`**: bundler novo e padrão.
- **`--tailwind`**: estilização rápida (você pode remover se preferir).
- **`--eslint`**: recomendado para padronização.

Entre na pasta e instale o axios com versão fixa:

```bash
cd crud-products
npm install axios@1.15.2
```

### Passo 2 — Configurar variável de ambiente

Crie o arquivo `.env.local` na raiz do projeto (esse arquivo **não** deve ser commitado):

```env
# .env.local
CRUDCRUD_API_BASE_URL=https://crudcrud.com/api/SUA_CHAVE_UNICA
```

Substitua `SUA_CHAVE_UNICA` pela chave que o site crudcrud.com fornece.

> **Atenção**: como não usamos o prefixo `NEXT_PUBLIC_`, essa variável fica disponível **apenas no servidor** — exatamente o que queremos, pois as chamadas ao crudcrud são feitas em Server Components e Server Actions.

### Passo 4 — Cliente HTTP com axios

Este arquivo roda **apenas no servidor** (nunca é importado por Client Components).

```js
// src/lib/products-api.js
import 'server-only';
import axios from 'axios';

const baseURL = process.env.CRUDCRUD_API_BASE_URL;

if (!baseURL) {
  throw new Error('CRUDCRUD_API_BASE_URL não configurada em .env.local');
}

export const api = axios.create({ baseURL });

export async function listProducts() {
  const { data } = await api.get('/products');
  return data;
}

export async function getProduct(id) {
  const { data } = await api.get(`/products/${id}`);
  return data;
}

export async function createProduct(payload) {
  const { data } = await api.post('/products', payload);
  return data;
}

export async function updateProduct(id, payload) {
  // crudcrud exige enviar o payload sem _id no update.
  const { _id, ...rest } = payload;
  await api.put(`/products/${id}`, rest);
}

export async function deleteProduct(id) {
  await api.delete(`/products/${id}`);
}
```

> **`import 'server-only'`** avisa o Next.js para quebrar o build caso esse módulo seja importado de um Client Component — protege contra vazamento acidental de credenciais.

Se você não instalou `server-only` (é opcional), remova a primeira linha.

### Passo 5 — Server Actions

Toda mutação do CRUD é uma **Server Action**. Elas são chamadas diretamente pelo `action={}` do `<form>` ou pelo `onClick` de um Client Component.

```js
// src/actions/products.js
'use server';

import { revalidateTag } from 'next/cache';
import { redirect } from 'next/navigation';
import {
  createProduct as apiCreate,
  updateProduct as apiUpdate,
  deleteProduct as apiDelete,
} from '@/lib/products-api';

const TAG = 'products';

function parseFormData(formData) {
  const name = String(formData.get('name') ?? '').trim();
  const price = Number(formData.get('price'));

  if (!name) throw new Error('Nome é obrigatório.');
  if (Number.isNaN(price) || price < 0) throw new Error('Preço inválido.');

  return { name, price };
}

export async function createProductAction(formData) {
  const payload = parseFormData(formData);
  await apiCreate(payload);
  revalidateTag(TAG);
  redirect('/products');
}

export async function updateProductAction(id, formData) {
  const payload = parseFormData(formData);
  await apiUpdate(id, payload);
  revalidateTag(TAG);
  redirect('/products');
}

export async function deleteProductAction(formData) {
  const id = String(formData.get('id'));
  await apiDelete(id);
  revalidateTag(TAG);
}
```

> `revalidateTag('products')` invalida o cache da tag, forçando o Server Component da listagem a buscar dados atualizados na próxima navegação.

### Passo 6 — Listagem (Server Component)

```jsx
// src/app/products/page.js
import Link from 'next/link';
import ProductTable from '@/components/ProductTable';

// Força renderização dinâmica: a listagem sempre busca dados frescos a cada request.
// (Evita também falha no `next build` caso a chave crudcrud ainda não esteja configurada.)
export const dynamic = 'force-dynamic';

async function fetchProducts() {
  // `next: { tags: ['products'] }` permite invalidar o cache com `revalidateTag('products')`.
  const res = await fetch(`${process.env.CRUDCRUD_API_BASE_URL}/products`, {
    next: { tags: ['products'] },
  });

  if (!res.ok) {
    throw new Error(`Falha ao carregar produtos: ${res.status}`);
  }
  return res.json();
}

export default async function ProductsPage() {
  const products = await fetchProducts();

  return (
    <main className="mx-auto max-w-3xl p-6">
      <div className="mb-4 flex items-center justify-between">
        <h1 className="text-2xl font-bold">Produtos</h1>
        <Link
          href="/products/new"
          className="rounded bg-blue-600 px-4 py-2 text-white hover:bg-blue-700"
        >
          Novo Produto
        </Link>
      </div>

      <ProductTable products={products} />
    </main>
  );
}
```

> Note que usamos `fetch` (API web) diretamente aqui, em vez de `axios`, porque o `fetch` do Next.js se integra ao sistema de cache via `next.tags`. O `axios` fica reservado para as Server Actions.

### Passo 7 — Tabela de produtos

A tabela é um Server Component, mas o botão de excluir precisa disparar uma Server Action via formulário:

```jsx
// src/components/ProductTable.js
import Link from 'next/link';
import { deleteProductAction } from '@/actions/products';

export default function ProductTable({ products }) {
  if (!products || products.length === 0) {
    return <p className="text-gray-600">Nenhum produto cadastrado.</p>;
  }

  return (
    <table className="w-full border-collapse border border-gray-300">
      <thead>
        <tr className="bg-gray-100">
          <th className="border p-2 text-left">Nome</th>
          <th className="border p-2 text-left">Preço</th>
          <th className="border p-2 text-left">Ações</th>
        </tr>
      </thead>
      <tbody>
        {products.map((product) => (
          <tr key={product._id}>
            <td className="border p-2">{product.name}</td>
            <td className="border p-2">
              {Number(product.price).toLocaleString('pt-BR', {
                style: 'currency',
                currency: 'BRL',
              })}
            </td>
            <td className="border p-2 flex gap-2">
              <Link
                href={`/products/${product._id}/edit`}
                className="text-blue-600 hover:underline"
              >
                Editar
              </Link>
              <form action={deleteProductAction}>
                <input type="hidden" name="id" value={product._id} />
                <button
                  type="submit"
                  className="text-red-600 hover:underline"
                >
                  Excluir
                </button>
              </form>
            </td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

> **Nota:** o botão "Excluir" dentro de um `<form>` com `action={deleteProductAction}` funciona até mesmo sem JavaScript no cliente (progressive enhancement).

### Passo 8 — Formulário reutilizável (Client Component)

Precisamos de estado local para exibir mensagens de erro/sucesso — logo, `'use client'`.

```jsx
// src/components/ProductForm.js
'use client';

import { useActionState } from 'react';

/**
 * action recebe (prevState, formData) e deve retornar o próximo state.
 */
async function runAction(action, prevState, formData) {
  try {
    await action(formData);
    return { ok: true, error: null };
  } catch (err) {
    return { ok: false, error: err.message || 'Erro desconhecido' };
  }
}

export default function ProductForm({ action, initialData }) {
  const boundAction = (prevState, formData) => runAction(action, prevState, formData);
  const [state, formAction, pending] = useActionState(boundAction, {
    ok: false,
    error: null,
  });

  return (
    <form
      action={formAction}
      className="flex flex-col gap-3 max-w-sm"
    >
      <label className="flex flex-col gap-1">
        <span className="font-semibold">Nome</span>
        <input
          type="text"
          name="name"
          defaultValue={initialData?.name ?? ''}
          required
          className="border rounded p-2"
        />
      </label>

      <label className="flex flex-col gap-1">
        <span className="font-semibold">Preço</span>
        <input
          type="number"
          name="price"
          step="0.01"
          min="0"
          defaultValue={initialData?.price ?? ''}
          required
          className="border rounded p-2"
        />
      </label>

      <button
        type="submit"
        disabled={pending}
        className="rounded bg-green-600 px-4 py-2 text-white hover:bg-green-700 disabled:opacity-50"
      >
        {pending ? 'Salvando…' : 'Salvar'}
      </button>

      {state?.error && (
        <p className="text-red-600">{state.error}</p>
      )}
    </form>
  );
}
```

> `useActionState` é o hook do React 19 para rastrear o estado de uma Server Action (pendente, retorno, erros).

### Passo 10 — Página "Editar produto"

```jsx
// src/app/products/[id]/edit/page.js
import ProductForm from '@/components/ProductForm';
import { getProduct } from '@/lib/products-api';
import { updateProductAction } from '@/actions/products';
import { notFound } from 'next/navigation';

export default async function EditProductPage({ params }) {
  const { id } = await params;

  let product;
  try {
    product = await getProduct(id);
  } catch {
    notFound();
  }

  // A action precisa conhecer o id. Usamos .bind para pré-carregá-lo.
  const actionWithId = updateProductAction.bind(null, id);

  return (
    <main className="mx-auto max-w-3xl p-6">
      <h1 className="mb-4 text-2xl font-bold">Editar Produto</h1>
      <ProductForm action={actionWithId} initialData={product} />
    </main>
  );
}
```

> `action.bind(null, id)` é o padrão oficial do React para pré-injetar parâmetros em uma Server Action antes de ela receber `formData`.

### Passo 12 — Rodar e testar

```bash
npm run dev
```

Fluxos para validar:

1. Acesse `http://localhost:3000/products` — a lista aparece vazia (ou com produtos já criados).
2. Clique em **Novo Produto**, preencha o formulário, submeta. Você é redirecionado para `/products` e o produto aparece.
3. Clique em **Editar** de um produto, altere campos, submeta — o valor é atualizado.
4. Clique em **Excluir** — o produto some da lista.

> Se o crudcrud retornar `404 Not Found`, sua unique key expirou (o serviço expira em 24h). Gere uma nova e atualize `.env.local`, depois reinicie o `npm run dev`.

### Próximos passos

- **Mais cadastros** (clientes, fornecedores): replique a estrutura `src/actions/*.js` + `src/lib/*-api.js` + rota `src/app/<entidade>/*`.
- **Validação robusta**: use [Zod](https://zod.dev/) em Server Actions para validar entrada.
- **Feedback UX**: combine `useActionState` com [`useFormStatus`](https://react.dev/reference/react-dom/hooks/useFormStatus) para botões desabilitados e mensagens de sucesso/erro sofisticadas.
- **Autenticação**: veja `5.autenticacao_autorizacao.md` para proteger essas rotas.

### Conclusão

Você construiu um CRUD completo usando o **App Router**, **Server Components**, **Server Actions** e revalidação de cache por tag — o padrão recomendado do Next.js 16. A separação entre:

- `src/lib/*-api.js` (acesso ao backend),
- `src/actions/*.js` (Server Actions com `'use server'`),
- `src/components/*Form.js` (Client Components com `useActionState`),
- `src/app/*/page.js` (Server Components que renderizam a UI),

torna o projeto fácil de estender para outros recursos.
